题目如下
<img src="\assets\images\max squence.jpg" />

一般情况下，我们小白看到这道题的首先想法很可能（反正我是）是用一层i循环来表示子数列长度，再用一层j循环来表示子数列起始长度，最后用一层k循环表示子数列结束位置，记录最大和，从而用O(n3)的时间复杂度完成了此题^_^(超时警告)

而经过初次优化后，我们学会了用一层i循环来表示子数列起始位置，一层j循环来表示子数列结束位置(此循环记录最大和)，优化到了O(n2)的时间复杂度(可喜可贺)。可惜这样我们还是会超时的。

所以，我们要向时间复杂度为O(n)的方向优化，老师提醒我们，我们用一层是来记录起始位置，另一层记录结束位置的两层循环来表示子数列，我们能不能用一层循环来表示不同的子数列呢？

最后，我们老师用结束位置来表示子数列，但不是只表示一个子数列，而是表示所有以a[i]结尾的子数列，用b来表示以a[i-1]为结尾的子数列的最大和,从而给我们引入了dp的思想,若b>0，就与a[i]相加，反之b=a[i]（及时止损）。通过子数列的表示方法优化了时间。

## 代码

```c++
int main(){
	int b=0,mix=0;
	int a[]={-2,11,-4,13,-5,-2};
	int n=sizeof(a)/sizeof(int);
	for(int i=0;i<n;i++){
		if(b>0)
			b+=a[i];
		else
			b=a[i];
		if(b>mix)
			mix=b;
	}
	printf("%d\n",mix);
	return 0;
}

```

dp方程以a[i]结尾的最大子数列和b[i]=max{a[i],b[i-1]+a[i]};

最后老师还给了一种贪心的做法（让我们证明是错的），这里就不写了，老师说dp会访问每一个子数列，而贪心不会。
